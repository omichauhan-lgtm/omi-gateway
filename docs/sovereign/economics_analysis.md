## 3. Cost-Savings Mechanisms

### 3.1 Expected Utility Routing
Instead of routing blindly to premium frontier models (e.g. GPT-4o at $15/million tokens), OMI routes default queries to cheap Indic language models (e.g. Sarvam-1 at $1/million tokens). Only high-entropy or complex requests are escalated.

### 3.2 Semantic Cache Reuse
For citizen portals, up to 30% of requests are queries for identical policy guidelines or services. OMI captures these in a stateful local database, resolving them locally with zero downstream API costs.

### 3.3 Context compression
If a user prompt includes a long document context, OMI applies context compression, reducing context sizes by up to 3x before model submission, significantly reducing input token costs.
