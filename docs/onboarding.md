# OMI Developer Quickstart Guide (3-Minute Setup)

Welcome to OMI, the open-source **Sovereign Cognitive Reliability Infrastructure** for AI systems. This guide will get you set up to route requests and inspect system calibration in under three minutes.

---

## 🛠️ Step 1: Install Dependencies

Ensure you have Python 3.12+ installed. Clone the repository and install the dependencies:

```bash
git clone https://github.com/omichauhan-lgtm/omi-gateway.git
cd omi-gateway
pip install -r requirements.txt
```

---

## ⚙️ Step 2: Configure Environment

Set up your API keys in a `.env` file in the root directory. If you want to test the infrastructure without active API keys, you can enable **Mock Mode**:

```ini
# Add your keys (Optional if using MOCK mode)
OPENAI_API_KEY=your-openai-key
ANTHROPIC_API_KEY=your-anthropic-key

# Enable sandbox simulation
USE_MOCK_PROVIDERS=True
OMI_ADMIN_KEY=omi-pro-key-v1
```

---

## 🚀 Step 3: Boot the Gateway Server

Start the FastAPI application using `uvicorn`:

```bash
python -m uvicorn api.main:app --port 8000
```

The OMI Gateway is now listening at `http://localhost:8000`.

---

## 📡 Step 4: Route Your First Prompt

Submit a prompt to the orchestrator. OMI will automatically analyze task complexity, route to the most cost-efficient provider (e.g. `sarvam-1` or `gpt-4o` depending on mode and language), evaluate the output confidence, and log telemetry.

```bash
curl -X POST http://localhost:8000/generate \
-H "Content-Type: application/json" \
-d '{
  "prompt": "Evaluate Article 21 of the Indian Constitution in relation to privacy.",
  "mode": "balance",
  "workflow_id": "project-quickstart-01"
}'
```

---

## 📊 Step 5: Query Live Verification Telemetry

To verify the calibration quality, provider reliability, and economic cost savings generated by OMI, call the public evidence endpoints:

### 1. High-Level Summary
```bash
curl -X GET http://localhost:8000/public/evidence
```

### 2. Calibration Curve
Obtain the mathematical ECE and accuracy bins proving calibration alignment:
```bash
curl -X GET http://localhost:8000/public/evidence/calibration
```

### 3. Developer & Project Adoption
Review Phase G3 target completion rates (active projects, user telemetry):
```bash
curl -X GET http://localhost:8000/public/evidence/adoption
```
